# Introduction

Welcome to the official API documentation for the **CellPay WEB SDK**, a service provided by Cellcom **Private Limited ("CELLCOM")**. This documentation is designed to support developers and technical teams in integrating the CellPay payment infrastructure into their applications.

CellPay is a comprehensive digital payment platform operating under the domain **[cellpay.com.np](https://www.cellpay.com.np)**. It allows users to access a wide range of financial services in partnership with licensed banks, merchants, government institutions, and other financial entities in Nepal. With CellPay, users can conveniently link their bank accounts (referred to as “your account”) from partner banks (“your bank”) directly to their CellPay account.

CELLCOM has developed a suite of APIs that enable seamless integration of CellPay’s payment functionalities into third-party platforms, including websites, mobile applications, and internal systems used by partner organizations and merchants.

## Intended Audience

This documentation is intended for businesses, developers, and organizations seeking to incorporate digital payment solutions into their services. It is suitable for:

- Merchants aim to embed CellPay as a payment method in their platform
- System integrators building end-to-end digital payment capabilities
- Developers responsible for implementing transaction flows within applications

### Purpose of the Document

This document is designed for partner merchants and developers who intend to integrate the CellPay Payment System into their platforms. It serves as a comprehensive guide to support the planning, development, and implementation phases of the integration process. The documentation outlines technical specifications, integration procedures, transaction flows, validation rules, and sample request/response structures for the API. The goal is to provide all necessary information in a streamlined manner to accelerate integration efforts. While this document does not cover the full implementation lifecycle on the vendor’s side, it is a critical resource that simplifies and supports integration from a technical perspective. It ensures that developers can efficiently incorporate CellPay services into their applications with clarity and ease. Moreover, the document explains the transaction lifecycle, including optional verification procedures, along with complete API references for all endpoints supported by the CellPay Payment System.
